Camera, Movimentação e Inputs

Para o tratamento da camera era nescessário que a mesma fosse tridimensional e que sua movimentação fosse fácil. No site r3dux.org existe um tutorial com um exemplo muito bom que satisfaz estes requisitos. O código do tutotial era um pouco datado e não orientado a objetos, então este foi atualizado e melhorado.

Primeiramente foi nescessário atualizar a biblioteca responsável de gerenciar a tela e os inputs de teclado e mouse. A biblioteca utilizada é a GLFW. Para obter a versão mais recente (versão 3.0) e os recursos mais avançados desta bibioteca foi nescessário instala-la a partir de seu código fonte, devido a não existência de uma versão atualizada para o sistema operacional utilizado (Ubuntu 15.04). O código fonte e o processo de instalação manual estão descritos no site da GLFW.

Com as modificações feitas ao código foi possivel alcançar os requisitos de camera e movimentação da seguinte forma:

  • A câmera é uma projeção OpenGL Frustum
  • É possível a movimentação em todas as direções, usando-se:
    • W: Mover para frente
    • S: Mover para trás
    • A: Mover para esquerda
    • D: Mover para direita
    • Shift: Mover para baixo
    • Espaço: Mover para cima
    • Movimento do mouse: Mover orientação da câmera
  • São realizadas operações de Rotação e Translação utilizando os atributos da câmera
  • É possível obter fullscreen pressionando F1
